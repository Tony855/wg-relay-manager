# WireGuard 多中继增强脚本

这是一个用于管理 WireGuard 多中继服务器的增强脚本，它提供了一个命令行界面和一个可选的 Web 管理界面来简化规则的配置和监控。

详细安装说明请参阅 [`INSTALL.md`](./INSTALL.md)。

## 功能特性

*   **模块化设计**: 脚本被拆分为多个功能模块，易于维护和扩展。
*   **命令行管理**: 提供丰富的命令行选项来管理转发规则、系统配置等。
*   **Web 管理界面**: 一个基于 Flask 的轻量级 Web 界面，提供直观的规则管理、系统状态监控和日志查看。
*   **流量统计**: 实时监控每个转发规则的流量使用情况。
*   **Nginx & SSL**: 自动配置 Nginx 作为 Web 界面的反向代理，并支持 Let's Encrypt 自动申请和续订 SSL 证书。
*   **Systemd 服务**: 将 Web 界面配置为 Systemd 服务，实现开机自启动和后台运行。
*   **依赖自动安装**: 自动检测并安装所需的系统依赖和 Python 依赖。

## 快速开始

### 1. 克隆仓库

```bash
git clone https://github.com/Tony855/wg-relay-manager.git
cd wg-relay-manager
```

### 2. 使用 root 权限运行主脚本

```bash
sudo bash main.sh
```

脚本启动后，建议按以下顺序完成首次部署：

1. 选择 `1. 安装/更新 WireGuard 中继环境`
2. 选择 `4. 配置系统参数`
3. 选择 `3. 启动/停止/重启 Web 管理界面`
4. 如果需要 HTTPS，再执行 `重新配置 Nginx 和 SSL`

完整步骤、升级流程和常见问题请查看 [`INSTALL.md`](./INSTALL.md)。

## 命令行菜单

运行 `sudo bash main.sh` 后，您将看到一个交互式菜单，提供以下选项：

1.  **安装/更新 WireGuard 中继环境**: 执行系统依赖安装、内核参数配置、防火墙初始化等。
2.  **管理转发规则 (命令行)**: 允许您在命令行添加、编辑、删除、启用/禁用和查看转发规则。
3.  **启动/停止/重启 Web 管理界面**: 控制 Flask Web 应用的运行状态，并可重新配置 Nginx 和 SSL。
4.  **配置系统参数**: 修改中继名称、Web 管理员用户名和密码、公网接口等。
5.  **查看系统状态和日志**: 实时查看 CPU、内存、磁盘、流量等系统状态，以及脚本运行日志。
6.  **卸载脚本**: 彻底移除脚本及其所有相关配置和文件。
7.  **退出**

## Web 管理界面

安装完成后，您可以通过浏览器访问 `https://您的服务器IP` 或 `https://您的域名` (如果配置了 SSL) 来访问 Web 管理界面。

默认登录凭据 (可在安装时修改或通过菜单选项 4 修改):
*   **用户名**: `admin`
*   **密码**: `admin` (请务必修改为强密码)

Web 界面提供了以下功能：

*   **仪表盘**: 显示 CPU、内存、磁盘使用率、连接数、运行时间、网络带宽等实时系统状态。
*   **端口转发规则**: 直观地添加、编辑、启用/禁用、删除和查看所有转发规则，并显示实时流量统计。
*   **系统设置**: 修改中继名称、Web 管理员凭据等。
*   **系统日志**: 查看脚本的运行日志。

## 注意事项

*   **安全性**: 默认的 Web 管理员密码非常弱，请务必在安装后立即修改。
*   **防火墙**: 脚本会自动配置 `iptables` 规则。如果您使用其他防火墙 (如 `ufw`, `firewalld`)，请确保它们不会干扰脚本的 `iptables` 配置。
*   **域名和SSL**: 如果您希望使用 HTTPS 访问 Web 界面，请确保您的域名已正确解析到服务器 IP，并在安装过程中提供有效的邮箱地址以申请 Let's Encrypt 证书。
*   **依赖**: 脚本会自动安装 `jq`, `python3`, `python3-pip`, `nginx`, `certbot` 等依赖。
